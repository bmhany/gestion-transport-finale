Moteur de generation des trajets Automatique

1. Les sources d'information

AffectationBusLigne Horaire Trajet
Actif

Le moteur lit ces trois entites pour construire les trajets a venir de maniere fiable.

Le traitement est centralise dans gestion_transport/signals.py.

2. Quand la generation se lance

Nouvelle affectation Mise a jour horaire Resync manuelle
Evenements

Chaque changement met a jour automatiquement la planification, sans creation manuelle des trajets.

La vue resync_trajets permet un recalcul global si necessaire.

3. Comment les trajets sont calcules

Nettoyage futur Parcours affectations Creation par horaires
Pipeline
Etape Objectif
Nettoyage des trajets futurs Eviter les doublons avant regeneration.
Parcours des affectations Verifier les periodes valides bus-ligne.
Creation des trajets Generer les trajets date + horaire + sens.

4. Regles et impact utilisateur

Jours valides Bus valide a la date Affectations futures Date fin controlee
Exploitation
A retenir : toute modification d'affectation ou d'horaire est repercutee automatiquement dans les trajets.

Resultat: une planification coherente, visible dans l'interface sans intervention repetitive.
